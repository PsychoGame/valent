Description

ValentManager effectively represents an instance of Valent, including the available ValentChannelService implementations that provide ValentChannel instances passed to ValentDevice instances.

valent_manager_identify ()

void
valent_manager_identify (ValentManager *manager,
                         const char *uri);

Request a connection from the device at uri if given, otherwise ask each loaded ValentChannelService to identity itself on its respective network.

The uri argument is string in the form backend://address, where "backend" is the module name and "address" is some address, DBus path or serialized data the backend can use to contact the device. A complete URI for the Lan backend might look like lan://192.168.0.10:1716.

Signal Details

The “device-added” signal

void
user_function (ValentManager *manager,
               ValentDevice  *device,
               gpointer       user_data)

“device-added” is emitted when a new ValentDevice has been added to manager . This usually happens when a ValentChannelService emits “channel” with an unknown identity.

Note that the internal state of manager has already been updated when this signal is emitted and manager will hold a reference to device .

The “device-removed” signal

void
user_function (ValentManager *manager,
               ValentDevice  *device,
               gpointer       user_data)

“device-removed” is emitted when a ValentDevice has been removed from manager . This usually happens when a device becomes both disconnected and unpaired, but also happens during shutdown when valent_manager_stop() is called.

Note that the internal state of manager has already be updated when this signal is emitted, but the last reference to device won't be dropped until the last signal handler returns.

Description

ValentContactStoreProvider is an interface for sources that provide objects that implement ValentContactStore. The interface is very simple and only requires that returned objects are implementations of ValentContactStore.

valent_contact_store_provider_load_async ()

                               (ValentContactStoreProvider *provider,
                                GCancellable *cancellable,
                                GAsyncReadyCallback callback,
                                gpointer user_data);

Requests that the ValentContactStoreProvider asynchronously load any known players.

This should only be called once on an ValentContactStoreProvider. It is an error to call this function more than once for a single ValentContactStoreProvider.

ValentContactStoreProvider implementations are expected to emit the “store-added” signal for each contact they've discovered. That should be done for known players before returning from the asynchronous operation so that the contact manager does not need to wait for additional players to enter the "settled" state.

ValentDevice

The ValentDevice object represents a remote device such as a smartphone, tablet or desktop.

ValentDevice implements GActionGroup and GActionMap, while providing a GMenu. ValentDevicePlugin implementations can add GActions and GMenuItems to expose plugin activities.

Properties

Valent.Device:connected

The “connected” property indicates whether the device is connected or not. This means an active connection that has been authenticated as being from the correct device, identities exchanged, encrypted and any other steps appropriate for the connection type.

Valent.Device:data

The “data” property is the ValentData for the device.

Valent.Device:icon-name

An icon name string for the device type.

Valent.Device:id

A unique string for the device, usually the hostname.

Valent.Device:name

The user-visible label for the device.

Valent.Device:paired

Whether the device is paired with regard to the KDE Connect protocol, regardless of the underlying transport protocol (eg. Bluetooth).

Valent.Device:state

The state of the device.

Valent.Device:type

A string hint, roughly describing the type of the device. It’s generally only useful for things like selecting an icon since the device will describe its true capabilities by other means.

Signals

Valent.Device::plugin-added

The “plugin-added” signal is emitted when a supported plugin implementing the ValentDevicePlugin interface has been loaded by the PeasEngine.

Valent.Device::plugin-removed

The “plugin-removed” signal is emitted when a supported plugin implementing the ValentDevicePlugin interface has been unloaded from the PeasEngine.

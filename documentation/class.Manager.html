valent_manager_identify

Request a connection from the device at uri if given, otherwise ask each loaded ValentChannelService to identity itself on its respective network.

valent_manager_start

Load all the ValentChannelService implementations known to the ValentEngine, thereby allowing new connections to be opened.

valent_manager_stop

Unload all the ValentChannelService implementations loaded from the ValentEngine, thereby preventing any new connections from being opened.

valent_manager_unexport

Unexport all managed devices, then manager from DBus.

Properties

Valent.Manager:data

The ValentData for the manager.

Valent.Manager:id

The unique ID this device will identify as. If not given a random ID will be generated.

Signals

Valent.Manager::device-added

ValentManager::device-added is emitted when a new ValentDevice has been added to manager. This usually happens when a ValentChannelService emits ValentChannelService::channel with an unknown identity.

Valent.Manager::device-removed

ValentManager::device-removed is emitted when a ValentDevice has been removed from manager. This usually happens when a device becomes both disconnected and unpaired, but also happens during shutdown when valent_manager_stop() is called.
